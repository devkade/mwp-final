# 테스트 시나리오 01: 로그인

## 목적

사용자 인증 시스템이 정상적으로 동작하는지 확인

## 사전 조건

-   [ ] Django 서버 실행 중 (`python manage.py runserver`)
-   [ ] 테스트용 사용자 계정 존재

## 테스트 단계

### 1. 서버 토큰 인증 API 테스트

```bash
# 토큰 발급 요청
curl -X POST http://127.0.0.1:8000/api-token-auth/ \
  -H "Content-Type: application/json" \
  -d '{"username": "test", "password": "test123123"}'
```

**예상 결과:**

```json
{ "token": "abc123..." }
```

| 항목           | 예상                   | 실제 | 통과 |
| -------------- | ---------------------- | ---- | ---- |
| HTTP 상태 코드 | 200                    |      | [ ]  |
| 토큰 반환      | JSON에 token 필드 존재 |      | [ ]  |

### 2. 잘못된 자격증명 테스트

```bash
curl -X POST http://127.0.0.1:8000/api-token-auth/ \
  -H "Content-Type: application/json" \
  -d '{"username": "wrong", "password": "wrong"}'
```

| 항목           | 예상             | 실제 | 통과 |
| -------------- | ---------------- | ---- | ---- |
| HTTP 상태 코드 | 400              |      | [ ]  |
| 에러 메시지    | 인증 실패 메시지 |      | [ ]  |

### 3. Android 앱 로그인 테스트

1. 앱 실행
2. 로그인 화면에서 사용자명/비밀번호 입력
3. 로그인 버튼 클릭

| 항목             | 예상               | 실제 | 통과 |
| ---------------- | ------------------ | ---- | ---- |
| 로그인 화면 표시 | 정상 표시          |      | [ ]  |
| 올바른 자격증명  | 메인 화면으로 이동 |      | [ ]  |
| 잘못된 자격증명  | 에러 메시지 표시   |      | [ ]  |
| 빈 필드          | 입력 요청 메시지   |      | [ ]  |
